Module 1 — CPU scheduler
C11
Main language. Burst measurement, decision tree, sched_setattr calls, resource allocation graph for deadlock.
gcc -O2 -std=c11
getrusage + clock_gettime
POSIX calls to measure per-process CPU burst time. Training data for the decision tree model.
sys/resource.h, time.h
eBPF (libbpf)
Attach to futex tracepoint to detect deadlock wait-chains between processes. Zero overhead vs ptrace.
pacman -S libbpf
Module 2 — Memory manager
C + /proc parser
Parse /proc/[pid]/smaps, /proc/meminfo. Implement Markov chain in pure C — no external ML library.
stdlib.h, stdio.h
zram (kernel module)
Compressed swap in RAM. TAORM sets it up at startup: modprobe zram, set lz4 algorithm, swapon at highest priority.
modprobe zram (built-in)
madvise + mincore
Core syscalls — mincore() reads page residency, madvise(MADV_FREE/WILLNEED) acts on AI prediction.
sys/mman.h
Module 3 — Security monitor
eBPF via bcc
Primary tracer — attaches a tracepoint to sys_enter. Writes syscall ID + PID to a perf ring buffer. Near-zero overhead.
pacman -S bcc python-bcc
seccomp + prctl
Apply syscall filter to flagged processes. libseccomp makes writing filter rules in C clean and safe.
pacman -S libseccomp
scikit-learn (one-class SVM)
Threat classifier trained on normal syscall patterns. Exported as a Python subprocess — C daemon queries it over stdin/stdout pipe.
pacman -S python-scikit-learn
Module 4 — I/O manager
ioprio_set syscall
Directly sets I/O scheduling class (RT / BE / IDLE) and level per PID. The primary control knob for Module 4.
linux/ioprio.h
iproute2 (tc via netlink)
Traffic control for network QoS. TAORM uses AF_NETLINK socket to add/modify tc qdisc rules without shelling out.
pacman -S iproute2
io_uring
Async I/O ring — used to monitor I/O completion events across processes without blocking. Linux 5.1+, Arch has it by default.
liburing (pacman -S liburing)

Language choices
✓ Do this
Write the daemon, all modules, and AI models in C11
Use Python only for scikit-learn SVM and the curses dashboard
Use eBPF (libbpf / bcc) for syscall tracing
Use libseccomp for writing seccomp filters cleanly
Use liburing for async I/O monitoring
✗ Avoid this
Don't write a kernel module — too risky, not needed
Don't use TensorFlow or PyTorch — way too heavy for 12 GB
Don't use Go or Rust for the daemon — C gives direct syscall access
Don't use ptrace as primary tracer — too slow; eBPF is better
Don't shell out to run tools (e.g. os.system("ionice")) — use syscalls directly
AI model choices
✓ Right size for laptop
Decision tree (depth 8) for CPU — fits in 2 KB, < 0.2ms inference
3rd-order Markov chain for memory — < 100 KB, handles temporal patterns well
5-gram hash table for security baseline — O(1) lookup per syscall
One-class SVM (scikit-learn) for threat classification — < 500 KB model
Online linear regression for I/O — 6 floats, updates in nanoseconds
✗ Too heavy for this machine
Don't use LSTM / RNN — needs GPU or wastes too much CPU
Don't use transformer models — 100 MB+ just for the model file
Don't use random forest (100 trees) — overkill latency for OS decisions
Don't batch-retrain from scratch — use online/incremental updates only
Don't write models to disk every cycle — only on daemon shutdown
Arch-specific choices
✓ Arch-native approach
Log to journald via sd_journal_send — integrates with journalctl
Install as a systemd service with proper unit file
Use a pacman hook to restart after kernel upgrades
Read config from /etc/taorm/taorm.conf (standard Arch convention)
Store model state in /var/lib/taorm/ between reboots
✗ Non-Arch patterns
Don't hardcode Ubuntu paths (/etc/default/, apt-specific layouts)
Don't use SysV init scripts — Arch is systemd-only
Don't depend on AppArmor profiles — Arch doesn't ship AppArmor by default
Don't pin Python versions — Arch is rolling, use whatever python3 is current
Don't use snap or flatpak packages for dependencies
